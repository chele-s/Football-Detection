DEMO - Football Detection System
================================

REQUIREMENTS:
------------
- Windows 10/11
- Python 3.9, 3.10, or 3.11
- NVIDIA GPU with updated drivers (optional, recommended)


INSTALLATION:
------------
1. Open PowerShell in the project folder
   (Right-click folder > "Open in Terminal")

2. Run:
   .\setup_demo.ps1

3. If permission error appears, run first:
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser


CONFIGURATION:
-------------
Edit file: configs\stream_config.yml

Change line:
  input_url: '/content/football.mp4'

To your video path, example:
  input_url: 'C:/videos/match.mp4'

Or direct URL:
  input_url: 'https://example.com/video.mp4'


Edit file: configs\model_config.yml

Change:
  device: 'cuda'     # If you have NVIDIA GPU
  device: 'cpu'      # If no GPU available

If using custom model weights:
  path: 'models/your_model.pt'


EXECUTION:
---------
1. Open PowerShell in project folder

2. Run:
   .\run_demo.ps1

3. Open in browser:
   http://localhost:8554/stream.mjpg

4. To stop: Ctrl+C in terminal


SHARE OVER INTERNET (OPTIONAL):
-------------------------------
1. While run_demo.ps1 is running, open ANOTHER terminal

2. Activate environment:
   .\.venv\Scripts\Activate.ps1

3. Install ngrok:
   pip install pyngrok

4. Run:
   python setup_ngrok_tunnel.py

5. Copy the public URL shown
   Example: https://xxxx-xxxx.ngrok.io/stream.mjpg

6. Share URL (can be opened in any browser or VLC)


TROUBLESHOOTING:
---------------
- Error "Python not found":
  Install Python from https://www.python.org/downloads/
  Check "Add Python to PATH" during installation

- PowerShell permission error:
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

- Video won't open:
  Verify path in stream_config.yml is correct
  Use forward slashes / instead of \
  Example: C:/videos/video.mp4

- CUDA/PyTorch error:
  Change to device: 'cpu' in model_config.yml

- Port already in use:
  Change mjpeg_port in stream_config.yml to another number (e.g. 8555)
